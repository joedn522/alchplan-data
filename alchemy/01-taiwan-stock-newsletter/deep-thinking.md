## 1. 品牌定位與命名

### 1.1 定位語

**一句話定位：** 「用 FAANG 工程師的數據實驗，拆解台股投顧話術」

**延伸定位：**
- 不賣明牌、不喊進出場，只做可重現的數據實驗
- 把投顧老師常說的「法人買超就會漲」「KD 黃金交叉必買」拿出來用數據驗證
- 實驗可能失敗，失敗本身就是有價值的內容

### 1.2 命名建議

| 方案 | 名稱 | Substack URL | 優缺點 |
|------|------|-------------|--------|
| A | **台股實驗室** | `twstock-lab.substack.com` | 直白好懂，SEO 強，但略普通 |
| B | **數據打臉** | `data-slap.substack.com` | 記憶點強，有態度，但可能太挑釁 |
| C | **WSL 台股研究報** | `wsl-finlab.substack.com` | 與現有品牌 wslfinlab 一致，有辨識度 |
| **推薦 D** | **台股數據實驗室 by WSL FinLab** | `wsl-twstock.substack.com` | 結合品牌與主題，兼顧辨識度和 SEO |

**推薦方案 D 的理由：**
- `WSL FinLab` 已經是你的平台品牌，電子報是它的延伸
- 「台股數據實驗室」清楚傳達內容屬性
- 未來如果平台成長，品牌一致性很重要

### 1.3 品牌調性

| 維度 | 定義 |
|------|------|
| 語氣 | 理性但不冷冰冰，像跟朋友聊天解釋實驗結果 |
| 視覺 | 深色底 + 螢光綠/藍的數據圖表風（科技感 + 金融感） |
| 人設 | 「在矽谷寫了十年程式的工程師，回來用數據看台股」 |
| 禁忌 | 不喊明牌、不預測大盤點位、不用「飆股」「翻倍」等投顧用語 |

### 1.4 Tagline 選項

- 「每週一個數據實驗，破解台股迷思」
- 「不賣明牌，只做實驗」
- 「FAANG 工程師的台股數據筆記」

#### 第 2 篇：法人連續買超 5 天，然後呢？（免費摘要 + 付費完整）

**目的：** 第一個完整實驗，展示「數據打臉」的方法

```
大綱：
【免費部分】
1. 投顧常說：「法人連買 → 跟進」，真的嗎？
2. 實驗設計（用白話解釋）
   - 資料範圍：2014-2024 台股全部上市公司
   - 定義：外資/投信/自營商連續買超 N 天（N=3,5,10）
   - 觀察：買超結束後 5/10/20/60 個交易日的報酬率
3. 關鍵結論（2-3 個 bullet points + 1 張摘要圖）
4. 一句話總結

【付費部分】
5. 完整數據表格（分產業、分市值級距）
6. 不同 N 值的敏感度分析
7. 加入「買超金額」作為篩選條件後的變化
8. 與大盤報酬率的比較（超額報酬分析）
9. 這個實驗的局限性與延伸方向

寫作時間預估：AI 初稿 20 分鐘，數據查詢 15 分鐘，修改 30 分鐘
圖表：免費 2 張，付費 4 張
```

#### 第 3 篇：KD 指標到底有沒有用？（免費摘要 + 付費完整）

```
大綱：
【免費部分】
1. KD 是散戶最愛的指標，但用數據驗證過嗎？
2. 實驗設計
   - KD 黃金交叉（K 線突破 D 線）作為買進訊號
   - KD<20 黃金交叉 vs KD 在任意位置黃金交叉
   - 觀察持有 5/10/20 天的報酬分布
3. 結論：勝率、平均報酬率、最大虧損
4. 跟隨機買進比較

【付費部分】
5. RSI、MACD 的同樣分析（技術指標橫向比較）
6. 不同參數設定的影響（KD(9,3,3) vs KD(14,3,3)）
7. 加入成交量濾鏡後的變化
8. 「技術指標組合」有沒有比單一指標好
```

#### 第 4 篇：為什麼你的回測報酬率 100% 但實際交易賠錢（免費）

**目的：** 方法論教育文，完全免費作為引流

```
大綱：
1. 一個真實場景：朋友跟我炫耀他的回測績效
2. 回測常見的五個陷阱
   - 倖存者偏差（下市股票不在資料裡）
   - 前視偏差（用了未來才知道的資訊）
   - 過度擬合（參數調到完美符合歷史）
   - 忽略交易成本（手續費、滑價、衝擊成本）
   - 忽略流動性（小型股的成交量不夠）
3. 每個陷阱用台股真實案例說明
4. 簡單的 checklist：怎麼判斷一個回測結果可不可信
5. CTA：下一篇會做一個「正確回測」的完整示範
```

#### 第 5 篇：營收創新高 = 該買進？（免費摘要 + 付費完整）

```
大綱：
【免費部分】
1. 每月 10 號營收公布，PTT/群組就開始喊「營收創高」
2. 實驗：營收月增率 / 年增率創 N 個月新高後的股價表現
3. 關鍵發現（劇透：沒有你想的那麼好）

【付費部分】
4. 分產業的細分數據（電子 vs 傳產 vs 金融）
5. 「連續 N 個月營收成長」vs 「單月爆發」的差異
6. 搭配本益比高低的交叉分析
7. 什麼條件下營收訊號才有效
```

#### 第 6 篇：我怎麼設計一個不會自欺欺人的回測框架（付費）

**目的：** 第一篇純付費文章，測試付費意願

```
大綱：
1. 為什麼要自己寫回測而不用現成工具
2. 一個好的回測框架需要什麼
   - 資料清理（處理除權息、下市股票、股票分割）
   - 交易成本模型（台股手續費 0.1425%、交易稅 0.3%/0.15%）
   - 滑價模型（用 bid-ask spread 估算）
   - 部位管理（等權重 vs 市值加權 vs 風險平價）
   - Out-of-sample 測試（為什麼 in-sample 績效是垃圾）
3. Walk-forward analysis 的概念與實作思路
4. 我自己踩過的三個坑（真實案例）
5. Checklist：回測品質自我檢查清單

注意：這篇只講框架思維，不碰 WSL FinLab 的具體實作
```

#### 第 7 篇：台股除權息行情是真的嗎？（免費摘要 + 付費完整）

```
大綱：
【免費部分】
1. 每年 6-8 月就有人說「除權息行情」
2. 實驗：高殖利率股在除息前後的表現
3. 「填息」的機率到底多高？

【付費部分】
4. 不同殖利率區間的填息機率與天數
5. 「除息前 N 天買進」的最佳 N 是多少
6. 二代健保補充保費的影響（>2 萬的門檻）
7. ETF 除息 vs 個股除息的差異
```

#### 第 8 篇：散戶最常犯的三個數據分析錯誤（免費）

**目的：** 另一篇完全免費文，維持免費內容的品質

```
大綱：
1. 錯誤一：只看平均值不看分布
   - 案例：「某策略平均報酬 15%」但中位數是 -3%
   - 附報酬率分布圖
2. 錯誤二：混淆相關性與因果性
   - 案例：「台積電漲 → 大盤漲」不代表「買台積電就能預測大盤」
3. 錯誤三：沒有對照組
   - 案例：「我的選股策略今年賺 20%」但大盤漲了 25%
4. 怎麼避免這些錯誤（簡單原則）
5. CTA：訂閱付費版，看更多深度方法論
```

## 4. X / IG 社群圖卡策略

### 4.1 X（Twitter）策略

**帳號：** 與 WSL FinLab 同一帳號，或另開 `@wsl_twstock`

**發文類型與頻率：**

| 類型 | 頻率 | 範例 |
|------|------|------|
| 實驗結論圖卡 | 每週一（配合 Substack 發布） | 「KD 黃金交叉的勝率是 47%，你還在用嗎？」 |
| 數據 one-liner | 每週三、五 | 「過去 10 年，外資連買 5 天後 20 日報酬率中位數：+1.2%（大盤同期：+1.0%）」 |
| 迷思 vs 數據 | 每週二或四 | 「投顧說：營收創高就買進 → 數據說：60 日超額報酬 -0.3%」 |
| 互動提問 | 每兩週一次 | 「你最想讓我用數據驗證哪個台股『鐵則』？」 |
| Substack 連結推廣 | 每週一 | 配合文章發布 |

**X 發文原則：**
- 每則推文都要有一個「反直覺的數據點」
- 用數字開頭（數字比文字更抓眼球）
- 永遠附圖（純文字推文觸及率低 60%）
- 爭議性標題 OK，但結論要客觀
- 回覆留言時保持「數據說話」的態度，不爭論

**X 貼文模板：**

```
模板 A：打臉型
[迷思] → [數據真相]
📊 [關鍵數字]
完整實驗報告 👇 [Substack 連結]

模板 B：數據速報
台股冷知識 #[編號]
[一句話數據結論]
[附圖]

模板 C：互動型
如果 [某條件]，你會買進嗎？
A) 買  B) 不買  C) 看情況
（數據答案在留言區）
```

## 6. 法律注意事項

### 6.1 台灣證券投資相關法規

**核心法規：** 《證券投資信託及顧問法》

| 法規重點 | 對電子報的影響 | 應對方式 |
|----------|---------------|---------|
| 未經許可不得經營投資顧問業務（§107） | 不能直接給「買 XX 股票」的建議 | 只做數據實驗，不給個股買賣建議 |
| 不得對外表示其能保證獲利或分擔損失（§104） | 不能暗示「照做就能賺錢」 | 強調實驗性質，過去不代表未來 |
| 不得約定或提供特定利益或佣金（§90） | 不能跟券商分潤推薦開戶 | 避免券商合作推薦（或確認合法方式） |

**安全的內容範圍：**
- 歷史數據的統計分析 → OK
- 方法論的教學 → OK
- 「如果你在過去 N 年用 X 策略，結果是 Y」 → OK（歷史回測，不是建議）
- 「你應該買 XX 股票」 → 不行
- 「我的模型預測明天會漲」 → 不行

### 6.2 免責聲明（必須在每篇文章底部）

本電子報所有內容僅供教育與研究目的，不構成任何投資建議、推薦或招攬。

- 投資有風險，任何投資決策應由讀者自行判斷並承擔風險
- 作者可能持有文中提及的相關證券部位

本電子報作者並非合格之證券投資顧問，未持有投顧執照。

## 9. 保護策略

### 9.1 保護原則

```
可以分享的                     不可以分享的
─────────────                 ─────────────
✅ 實驗結論                    🚫 模型用了哪些 features
✅ 聚合統計數據                🚫 feature engineering 的細節
✅ 公開可得的方法論             🚫 模型架構（XGBoost? LSTM? Ensemble?）
✅ 回測的概念和框架             🚫 超參數設定
✅ 常見的陷阱和錯誤             🚫 訓練/驗證/測試的切分方式
✅ 績效的概略描述               🚫 即時的交易訊號
  （「年化報酬雙位數」）        🚫 完整的 feature importance
✅ 資料清理的一般方法           🚫 原始程式碼
✅ 某個因子「有效/無效」的結論  🚫 因子的具體組合權重
```

### 9.2 具體保護手法

#### 手法 1：時間延遲

- 實驗結果延遲 1-4 週發布
- 絕不發布「即時」或「本週」的交易觀點
- 例：3 月做的實驗，4 月才發布

#### 手法 2：聚合而非個股

- 發布「全市場統計」而非「這 10 檔股票符合條件」
- 例：「符合條件的股票平均報酬 X%」而不是「2330、2454 符合條件」
- 如果要舉個股案例，只用過去已結束的例子

#### 手法 3：方法論層級控制

```
Level 0（公開）：概念層
  「我們用機器學習模型分析台股」

Level 1（免費文章）：方法層
  「我們用了基本面、技術面、籌碼面的數據做分析」

Level 2（付費文章）：框架層
  「我們的回測框架包含 walk-forward analysis、交易成本模型等」

Level 3（永不公開）：實作層
  「我們用了 XYZ features，模型是 ABC，超參數是 123」
```

#### 手法 4：教「釣魚」不教「魚點」

- 教讀者「怎麼設計一個好的回測」而不是「我的回測怎麼做的」
- 教「怎麼評估一個因子有沒有效」而不是「我用了哪些因子」
- 教「避免過度擬合的方法」而不是「我的模型怎麼避免過度擬合」

#### 手法 5：反向利用「AI 時代大家都會」的焦慮

這個焦慮其實是內容素材：
- 「AI 時代，散戶還有機會嗎？」 → 寫成一篇文章
- 「量化交易門檻在降低，但 alpha 在哪裡？」 → 討論 alpha decay
- 教方法論反而建立權威，因為你展示的是「你比別人走得更前面」

### 9.3 如果有人問太深入的問題

| **時間不夠**：工作忙碌時無法維持週更 | 高 | 高 | 預寫 2-3 篇存稿；自動化工作流減少每週時間 |
| **法律風險**：被認定為無照投顧 | 低 | 極高 | 嚴格遵守免責聲明、不給個股建議、必要時諮詢律師 |
| **沒人看**：內容好但沒有觸及 | 中 | 中 | 持續優化 X/IG 策略；靠 1-2 篇爆文引流 |
| **付費轉換低**：人看但不付費 | 中 | 中 | 調整免費/付費的分界；確保付費內容有明確差異化 |
| **被抄襲**：別人複製你的實驗方法 | 中 | 低 | 抄方法沒關係（方法是公開知識），抄不走模型核心 |
| **模型洩露**：不小心在文章中暴露太多 | 低 | 極高 | 每篇發布前用 checklist 檢查；設定「禁止詞清單」 |
| **市場環境差**：台股長期下跌，無人關注 | 低 | 中 | 熊市反而是好內容（「下跌時哪些策略有效？」） |
| **Substack 平台風險**：政策改變或衰落 | 低 | 中 | 定期匯出訂閱者 email 名單（Substack 允許） |
| **AI 競爭**：人人都能做量化分析 | 高 | 中 | 見 9.3 — 這反而是內容素材；你的優勢是「已經做了+有結果」 |

### 10.2 實驗題庫（解決內容枯竭）

持續維護一份「待做實驗」清單，確保至少有 20 個以上的主題待寫：

**技術面系列：**
- [ ] MACD 黃金交叉的勝率
- [ ] 布林通道突破的回測
- [ ] 均線策略（5MA/20MA/60MA 交叉）到底有多少用
- [ ] RSI 超買超賣的反轉機率
- [ ] 成交量爆量後的走勢統計

**籌碼面系列：**
- [ ] 融資/融券餘額變化與股價的關係
- [ ] 主力進出的預測力
- [ ] 外資 vs 投信，誰的勝率高？
- [ ] 散戶指標（融資維持率）的逆勢效果
- [ ] 借券賣出與股價的關係

**基本面系列：**
- [ ] 本益比低估股的長期表現

- [ ] 高 ROE 策略在台股的表現
- [ ] 股利殖利率選股的風險

**市場行為系列：**

### 10.3 發布前安全 Checklist

每篇文章發布前必須檢查：

```
□ 是否包含個股買賣建議？→ 有則刪除
□ 是否暴露模型 features？→ 有則改為通用描述
□ 是否提及模型架構或超參數？→ 有則刪除
□ 是否包含即時或近期的交易訊號？→ 有則改為歷史數據
□ 免責聲明是否在文末？→ 沒有則加上
□ 利益衝突是否揭露？→ 沒有則加上
□ 數據是否正確（抽查 2-3 個數字）？→ 錯誤則修正
□ 圖表是否有清楚的標題和軸標籤？→ 沒有則補上
□ 是否有禁止詞（飆股、翻倍、必漲、保證獲利）？→ 有則替換
□ 語氣是否符合品牌調性？→ 不符則調整
```
