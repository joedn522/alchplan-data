# 深度研究報告：OpenClaw Skills vs MCP vs Claude Code

## 1. 核心問題：既然有了 Skill，為什麼還要 MCP？
您在筆記中提到：「不是只要有 skill 萬事都 OK 嗎？」
這是一個非常銳利的觀察。簡單來說，**Skill 是 OpenClaw 的原生插件，而 MCP 是「工業標準接口」。**

### Skill 的局限性
*   **封閉性**：Skill 通常是為 OpenClaw 量身打造的（JavaScript/TypeScript），很難直接拿到別的 AI 平台用。
*   **維護成本**：每出一個新工具（例如新的股市 API），社群就要寫一個新的 Skill。

### MCP (Model Context Protocol) 的優勢
*   **通用性**：MCP 是 Anthropic 推出的標準。現在網路上已經有成千上萬個現成的 **MCP Servers**（例如 Google Drive, Slack, GitHub, SQLite）。
*   **隨插即用**：有了 `mcp-adapter`，OpenClaw 就不需要再寫「Google Drive Skill」，而是直接「掛載」Google 官方或社群維護的 MCP Server。
*   **痛點解決**：解決了「擴充速度跟不上需求」的問題。您可以直接把 Claude Desktop 能用的工具，一秒搬給 OpenClaw 用。

| 維度 | **OpenClaw (Pi)** | **Claude Code / Codex** |
| :--- | :--- | :--- |
